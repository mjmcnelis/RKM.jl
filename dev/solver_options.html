Solver options

The ODE evolution function evolve_ode requires a set of solver options, which are stored in the struct SolverOptions. Here is an example set of all the available options:

options = SolverOptions(;
    method = RungeKutta4(),
    adaptive = Fixed(),
    timer = TimeLimit(),
    state_jacobian = FiniteJacobian(),
    root_finder = Newton(),
    eigenmax = NoEigenMax(),
    sensitivity = NoSensitivity(),
    interpolator = NoInterpolation(),
    save_solution = true
    save_time_derivative = false
    show_progress = false
    time_subroutine = false
    precision = Float64
)

We provide an overview of each solver option:

1. method

You must specify the ODE method used for the time evolution. The solver supports two classes of ODE methods: Runge–Kutta and linear multistep. - Euler1() - sgdgs

RKM.SolverOptions — Type
struct SolverOptions{T<:AbstractFloat}

List of options for the ODE solver.

  • method::RKM.ODEMethod: ODE solver method

  • adaptive::RKM.AdaptiveTimeStep: Adaptive time step method

  • timer::TimeLimit: Timer for ODE solver

  • state_jacobian::RKM.JacobianMethod: State Jacobian method for implicit ODE solvers or sensitivity analysis

  • root_finder::RKM.RootFinderMethod: Root finder for implicit ODE solvers

  • eigenmax::RKM.EigenMaxMethod: Max eigenvalue method for implicit ODE solvers

  • sensitivity::RKM.SensitivityMethod: Sensitivity method

  • interpolator::RKM.Interpolator: Interpolation method for dense output

  • save_solution::Bool: Store numerical solution (time and state variables)

  • save_time_derivative::Bool: Additionally output time derivatives

  • show_progress::Bool: Display progress meter

  • time_subroutine::Bool: Estimate runtime of core subroutines (e.g. function evaluations)

  • precision::Type{T} where T<:AbstractFloat: Numerical precision of the solver and solution
